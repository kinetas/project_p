# 가치투자 종목 발굴 서비스 — 기획 정리

> 마지막 업데이트: 2025.06.09  
> 상태: ✅ 결정 완료 항목 / 🔲 미결 항목 구분

---

## 1. 핵심 설계 원칙

> **페이지마다 딱 하나의 질문에 답한다**

| 페이지 | 사용자가 묻는 질문 | 보여주는 지표 |
| --- | --- | --- |
| 메인/목록 | 이 종목 싸 보이나? | PER · PBR · ROE · 부채비율 |
| 상세 상단 | 지금 이 주식 어떤 상태인가? | 6개 핵심 지표 요약카드 |
| 상세 하단 | 이 기업이 지속적으로 좋은 기업인가? | 최근 5년 추이 (차트 + 테이블) |

---

## 2. 지표 정의

각 지표는 아래 세 가지 질문 중 하나에 대답합니다.

- **① 저평가** — 이 주식이 싸게 거래되고 있는가?
- **② 수익성** — 이 기업은 돈을 잘 버는가?
- **③ 안전성** — 이 기업은 재무적으로 안전한가?

| 지표 | 분류 | 가치투자에서의 의미 |
| --- | --- | --- |
| PER | ① 저평가 | 이익 대비 주가. 낮을수록 싸게 사는 것. 핵심 중의 핵심 |
| PBR | ① 저평가 | 순자산 대비 주가. 1 미만이면 청산가치보다 싼 것 |
| EPS | ② 수익성 | PER 계산의 기반. 주당 얼마 버는지 |
| BPS | ① 저평가 | PBR 계산의 기반. 주당 순자산 |
| ROE | ② 수익성 | 자본 대비 얼마나 버나. 높을수록 좋은 기업 |
| 영업이익률 | ② 수익성 | 본업에서 얼마나 남기나. 지속적인 경쟁력 판단 |
| 부채비율 | ③ 안전성 | 빚이 너무 많으면 위기 때 쓰러짐. 가치주 필수 체크 |
| 매출액 추이 | ② 수익성 | 성장하는 기업인지 쪼그라드는 기업인지 판단 |
| 자산/부채/자본 | ③ 안전성 | 재무건전성의 원재료 |
| 배당수익률 | ① 저평가 | 실제 현금 수익. 추가 API 필요하여 후순위 처리 |

## 3. 페이지 구성

### 3-1. 메인/목록 페이지 ✅

**결정: 메인과 목록을 처음에는 분리해서 개발 → 이후 합치기**

분리해서 시작하는 이유:
- 팀원 간 역할 분담이 페이지 단위로 명확해짐
- 잘 만든 두 컴포넌트를 합치는 건 쉬움 (반대 방향은 어려움)
- React Router 기반이므로, 합치는 작업은 프론트엔드 컴포넌트 재배치만으로 가능

**최종 목표 화면 구조:**

```
[검색창]
[필터바]

───── 가치주 TOP10 / 저PER TOP10 / 고ROE TOP10 (탭 전환) ─────

─────────────── 전체 종목 리스트 ────────────────────────
종목명 | 현재가 | 시가총액 | PER | PBR | ROE | 부채비율
```

**목록 컬럼 (고정 6개):**

| 컬럼 | 분류 | 비고 |
| --- | --- | --- |
| 현재가 | 기본 정보 | |
| 시가총액 | 기본 정보 | |
| PER | ① 저평가 | |
| PBR | ① 저평가 | |
| ROE | ② 수익성 | |
| 부채비율 | ③ 안전성 | 배당수익률 대신 채택 — 가치 함정 종목 필터링 목적 |

> 배당수익률은 상세 페이지에서 확인. 목록에서는 안전성 지표 우선.

### 3-2. 상세 페이지 ✅

#### 상단 — 요약 카드

```
종목명 (ex. 삼성전자)
종목코드 (ex. 005930)

| 항목    | 값 |
| 현재가   |   |
| 시가총액  |   |
| PER     |   |
| PBR     |   |
| ROE     |   |
| 배당수익률 |   |
```

> 주가 차트는 우선순위 후순위로 미룸

---

#### 중단 — 기업 정보 + 가치 지표

```
[기업 정보]
| 항목   | 값     |
| 업종   |       |
| 시장구분 | KOSPI |
| 상장일  |       |
| 대표자  |       |

[가치 지표]
| 항목      | 값 | 설명        |
| EPS      |   | 주당순이익    |
| BPS      |   | 주당순자산    |
| ROE      |   |            |
| 부채비율   |   | 안전성 판단   |
| 영업이익률  |   | 수익성 판단   |
```

> PER · PBR은 상단 요약카드에 이미 있으므로 중단에서 제거

---

#### 하단 — 재무제표 (최근 5년)

**차트 3개:**

| 차트 | 담는 의미 |
| --- | --- |
| 매출액 + 영업이익 (묶음) | 성장성 + 수익성 |
| ROE 추이 | 수익성의 지속성 |
| 부채비율 추이 | 안전성 변화 |

> 자산/부채/자본을 각각 차트로 만들면 정보 과부하. 부채비율 하나로 세 값의 관계를 요약.

## 4. 데이터 흐름

```
코스피 + 코스닥 전체 종목
        ↓
    종목코드 확보 (KRX)
        ↓
  DART corp_code 매핑
        ↓
    재무제표 수집 (DART)
        ↓
  PER / PBR / ROE 계산
        ↓
      DB 저장
        ↓
      필터링
```

## 6. 미결 사항

| 항목 | 내용 | 우선순위 |

| 🔲 가치주 종합점수 계산 방식 | PER+PBR+ROE+부채비율 → 백분위 환산 후 합산 방식 유력. 가중치 팀 합의 필요 | 높음 |
